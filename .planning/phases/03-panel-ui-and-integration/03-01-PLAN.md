  <name>Task 1: Create tree.lua -- tree state model and renderer</name>
  <files>lua/ts-explorer/tree.lua</files>
  <action>
Create `lua/ts-explorer/tree.lua` implementing a flat rendering model for TypeNode trees.

**Data model:**
- Tree state is `{ root = TypeNode_table, expanded = {} }` where `expanded` is a set of path strings
- Path format: `"0"` for root, `"0.1"` for second child of root, `"0.1.3"` for fourth child of that, etc.

**Public API:**

1. `M.new(type_node)` -- Create tree state from a TypeNode Lua table. Calls `expand_default` automatically.

2. `M.expand_default(tree_state)` -- Set root path `"0"` as expanded (depth 1: root expanded, children visible but collapsed). Per user decision: default expand depth is 1 level.

3. `M.render(tree_state)` -- Walk the tree and produce `{ lines = string[], line_map = path[] }`.
   - `lines`: Array of rendered text lines for `nvim_buf_set_lines`
   - `line_map`: Array of path strings, one per line, for mapping cursor line to tree node

4. `M.toggle(tree_state, line_number)` -- Toggle expand/collapse for the node at the given 1-based line number. Only acts on nodes with children. Returns true if state changed.

5. `M.expand_recursive(tree_state, line_number)` -- Expand the node at line_number and all its descendants recursively. Returns true if state changed.

6. `M.collapse_recursive(tree_state, line_number)` -- Collapse the node at line_number and all its descendants recursively. Returns true if state changed.

7. `M.node_at_line(tree_state, line_number)` -- Return the path string for the given 1-based line number (from last render's line_map), or nil.

**Rendering rules (per user locked decisions):**
- Indent: 2 spaces per depth level
- Markers: `▸ ` for collapsed node with children, `▾ ` for expanded node with children, `  ` (2 spaces) for leaf nodes (alignment)
- Node format: `{indent}{marker}{prefix}{name}{suffix}: {typeString}`
- `prefix`: `"readonly "` if `node.readonly` is truthy, else `""`
- `suffix`: `"?"` if `node.optional` is truthy, else `""`
- Function return type child nodes have name `"[return]"` (already set by sidecar)
- Union branches and function params are children (already structured by sidecar)
- Root shows its own `name: typeString` like any other node

**Internal `_render_node(tree_state, node, path, depth, lines, line_map)`:**
- Build the line string per rules above
- Append to `lines` and `line_map`
- If node has children AND path is in `expanded` set, recurse into each child with path `parent_path .. "." .. (i-1)` and `depth + 1`

**For `expand_recursive`:** Walk the subtree from the target node, adding all paths that have children to the expanded set.

**For `collapse_recursive`:** Remove the target path and all paths starting with `target_path .. "."` from the expanded set.

Store last render result in tree_state as `tree_state._last_render` so `node_at_line` can reference it without re-rendering.
