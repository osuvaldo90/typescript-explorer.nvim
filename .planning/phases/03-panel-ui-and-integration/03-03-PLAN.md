    - Test: resolveAtPosition on `EventBus` class declaration returns a valid TypeNode without throwing
    - Test: typeToString fallback produces a non-empty string (not crash) for deeply recursive types
    - Test: Existing cycle detection tests still pass (tree fixture)
  </behavior>
  <action>
    Create a `safeTypeToString` wrapper function in type-walker.ts that replaces all direct `checker.typeToString(type, undefined, NoTruncation)` calls. The wrapper should:

    1. First try `checker.typeToString(type, undefined, ts.TypeFormatFlags.NoTruncation)` in a try-catch
    2. On RangeError (stack overflow), fall back to `checker.typeToString(type)` (default truncation) in another try-catch
    3. On second failure, return the type's symbol name or `"[complex type]"` as last resort

    Also add a `maxDepth` parameter to `walkType` (default 15). When depth exceeds maxDepth, return a node with kind "timeout" and typeString "[max depth exceeded]" instead of recursing further. This prevents the stack overflow from occurring in the first place by stopping recursion before TS internals overflow.

    Update the walkType signature to accept depth:
    ```typescript
    export function walkType(
      checker: ts.TypeChecker,
      type: ts.Type,
      name: string,
      visited: Set<number>,
      startTime: number,
      timeoutMs: number,
      depth?: number,
    ): TypeNode
    ```

    All recursive calls to walkType must pass `(depth ?? 0) + 1`.

    Add a test fixture or use the existing test.ts `EventBus` pattern. Create `sidecar/test-fixtures/classes.ts` with a generic class that has a Map property (triggers the recursion). Write a test that resolves the class instance and asserts no RangeError is thrown and result.node is non-null.

    Replace all bare `checker.typeToString(...)` calls with `safeTypeToString(checker, type)` throughout the file (there are ~3 call sites: the main typeString computation, union branch names, and the circular marker).
  </action>
  <verify>
    <automated>cd sidecar && npx tsx --test src/services/type-walker.test.ts 2>&1 | tail -20</automated>
  </verify>
  <done>

    - Existing 14 type-walker tests still pass
    - New tests for recursive class types pass
    - maxDepth guard prevents unbounded recursion
  </done>
</task>
