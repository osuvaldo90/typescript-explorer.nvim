  <behavior>
    ## Object types (TRES-02)
    - Given a symbol of type `{ id: number; name: string }`, walker returns kind "object" with children for each property
    - Each property child has name matching the property name and typeString matching the property type

    ## Union types (TRES-03)
    - Given `string | number`, walker returns kind "union" with two children (one per branch)
    - Per user decision: branches are flat direct children of the union node

    ## Intersection types (TRES-04)
    - Given `{ a: string } & { b: number }`, walker returns kind "object" (merged result) with children for both `a` and `b`
    - Per user decision: use `checker.getApparentType()` to get merged view, not separate intersection constituents

    ## Function types (TRES-05)
    - Given `(name: string, age: number) => string`, walker returns kind "function" with parameter children and a "returns" child
    - Parameter children have names matching parameter names

    ## Array types (TRES-06)
    - Given `string[]`, walker returns kind "array" with one "element" child of kind "primitive"
    - Given `[string, number]`, walker returns kind "tuple" with positional children `[0]`, `[1]`

    ## Generic types (TRES-07)
    - Given `Pair<string, number>` at usage site, the resolved type shows the concrete types, not generic parameters

    ## Optional marker (TRES-08)
    - Given property `email?: string`, the TypeNode has `optional: true`

    ## Readonly marker (TRES-09)
    - Given property `readonly id: number`, the TypeNode has `readonly: true`

    ## Cycle detection
    - Given `type Tree = { left: Tree; right: Tree }`, the walker expands one level then produces kind "circular" with `[circular: Tree]` typeString
    - The walker does not stack overflow

    ## Timeout (SIDE-06)
    - Given a timeout of 1ms (artificially low), the walker returns kind "timeout" nodes for unresolved parts
    - Per user decision: partial results returned, not error response
  </behavior>
  <implementation>
Create `sidecar/src/services/type-walker.ts` exporting:

1. `resolveAtPosition(filePath: string, position: number, timeoutMs?: number): ResolveResult` -- entry point that:
   - Calls `getLanguageService(filePath)` to get the service
   - Gets Program, SourceFile, TypeChecker
   - Finds the innermost AST node at the position
   - Gets the symbol/type at that node
   - Calls `walkType` with a new visited set and start time
   - Returns `{ node }` or `{ node: null }` if no symbol found
   - Default timeout: 5000ms

2. `walkType(checker, type, name, visited, startTime, timeoutMs)` -- recursive walker:
   - Check timeout first: if `Date.now() - startTime > timeoutMs`, return `{ kind: "timeout", name, typeString: "[resolution timeout]" }`
   - Check cycle: if `(type as any).id` is in `visited` Set, return `{ kind: "circular", name, typeString: "[circular: {typeName}]" }`
   - Add type.id to visited
   - Get typeString via `checker.typeToString(type, undefined, ts.TypeFormatFlags.NoTruncation)`
   - Check type kinds in this order (order matters for correct classification):
     a. `type.isUnion()` -- map branches to children, each branch gets `new Set(visited)`
     b. `type.isIntersection()` -- use `checker.getApparentType(type).getProperties()` for merged view, return as kind "object"
     c. `checker.isTupleType(type)` -- get type args, children named `[0]`, `[1]`, etc.
     d. `checker.isArrayType(type)` -- get type args[0], child named "element"
     e. `type.getCallSignatures().length > 0` -- params + "returns" child
     f. `type.flags & ts.TypeFlags.Object && type.getProperties().length > 0` -- object with property children
     g. `type.flags & (ts.TypeFlags.Enum | ts.TypeFlags.EnumLiteral)` -- kind "enum"
     h. `type.isLiteral()` -- kind "literal"
     i. Default: kind "primitive"
   - Delete type.id from visited before returning (backtrack)

3. `walkSymbol(checker, symbol, visited, startTime, timeoutMs)` -- helper for property symbols:
   - Gets type via `checker.getTypeOfSymbol(symbol)`
   - Calls `walkType` with the type
   - Checks `symbol.flags & ts.SymbolFlags.Optional` for optional marker
   - Checks `ts.getCombinedModifierFlags(declarations[0]) & ts.ModifierFlags.Readonly` for readonly marker
   - Extracts source location from declarations for `sourcePath`/`sourceLine`

Create additional fixture files as needed:
