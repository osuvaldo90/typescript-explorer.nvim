  <action>
Create `sidecar/src/types.ts` with the following exports:

1. `TypeKind` -- union type of all node kinds: `"object" | "union" | "intersection" | "function" | "array" | "tuple" | "primitive" | "literal" | "enum" | "circular" | "timeout"`

2. `TypeNode` -- the core tree node interface:
   - `kind: TypeKind`
   - `name: string` -- property name, parameter name, or index
   - `typeString: string` -- full TS compiler type string (NoTruncation)
   - `optional?: boolean` -- true for optional properties (TRES-08)
   - `readonly?: boolean` -- true for readonly properties (TRES-09)
   - `sourcePath?: string` -- declaration file path (for v2 NAV-01)
   - `sourceLine?: number` -- declaration line number (for v2 NAV-01)
   - `children?: TypeNode[]` -- child nodes (properties, union branches, params, etc.)

3. `ResolveParams` -- request params interface:
   - `filePath: string` -- absolute path to the TypeScript file
   - `position: number` -- byte offset cursor position in the file

4. `ResolveResult` -- response result interface:
   - `node: TypeNode | null` -- the resolved type tree, null if no symbol found

  <action>
Create `sidecar/src/services/language-service.ts`:

Export a `getLanguageService(filePath: string)` function that:

1. Resolves the absolute path of `filePath`
2. Uses `ts.findConfigFile(path.dirname(filePath), ts.sys.fileExists, "tsconfig.json")` to discover the tsconfig
3. If found: `ts.readConfigFile` + `ts.parseJsonConfigFileContent` to get compiler options and file list
4. If not found: use sensible defaults (`target: ES2022`, file list = `[filePath]`)
5. Creates a `ts.LanguageServiceHost` implementation:
   - `getScriptFileNames()` -- returns parsed file list (ensure the requested filePath is always included even if not in tsconfig include)
   - `getScriptVersion()` -- returns "0" (we don't need file watching for v1)
   - `getScriptSnapshot()` -- reads file via `fs.readFileSync`, returns `ts.ScriptSnapshot.fromString()`
   - `getCurrentDirectory()` -- project root (tsconfig dir or file dir)
   - `getCompilationSettings()` -- parsed compiler options
   - `getDefaultLibFileName()` -- `ts.getDefaultLibFilePath(options)`
   - Delegate `fileExists`, `readFile`, `readDirectory`, `directoryExists`, `getDirectories` to `ts.sys`
6. Calls `ts.createLanguageService(host, ts.createDocumentRegistry())`
7. Caches by tsconfig path (or project dir for no-tsconfig case) using a `Map<string, ts.LanguageService>`

Write tests in `sidecar/src/services/language-service.test.ts` using `node:test` that verify all behaviors listed above. Use the test-fixtures/ directory for the fixture file paths.
