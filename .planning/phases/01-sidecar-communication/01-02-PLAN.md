  <action>
Create all Lua modules following the research patterns. Build bottom-up: config -> log -> rpc -> sidecar -> init -> plugin entry.

1. **lua/ts-explorer/config.lua** -- Configuration with defaults:
   - `M.defaults = { sidecar = { max_restarts = 3 }, log = { level = "error" } }`
   - `M.values = {}` (merged config after setup)
   - `M.setup(opts)` merges user opts with defaults via `vim.tbl_deep_extend("force", M.defaults, opts or {})`
   - `M.get()` returns M.values

2. **lua/ts-explorer/log.lua** -- Logging:
   - `M.debug(...)` -- only logs if config log level is "debug", uses `vim.notify` with DEBUG level
   - `M.error(...)` -- always logs via `vim.notify` with ERROR level, prefixed "[ts-explorer]"
   - `M.on_stderr(data)` -- receives stderr lines from sidecar, forwards to debug log

3. **lua/ts-explorer/rpc.lua** -- NDJSON message protocol:
   - `local buffer = ""` for partial line assembly
   - `local pending = {}` table mapping id -> callback
   - `local next_id = 0` counter
   - `M.on_data(data)` -- receives the raw data table from jobstart on_stdout. Implements line buffering per research Pattern 4: first element appends to buffer, subsequent elements indicate newline boundaries. Process complete lines through `_process_line()`.
   - `M._process_line(line)` -- skip empty lines, `pcall(vim.json.decode, line)`, if valid and has id with pending callback, invoke callback(msg.error, msg.result) and remove from pending.
   - `M.request(method, params, callback)` -- increments next_id, stores callback in pending[id], encodes `{id=id, method=method, params=params}` with `vim.json.encode()` + "\n", sends via `vim.fn.chansend(require("ts-explorer.sidecar").get_job_id(), msg)`.
   - `M.reset()` -- clears buffer, pending, resets next_id (called on sidecar restart).

4. **lua/ts-explorer/sidecar.lua** -- Process lifecycle:
   - `local job_id = nil`
   - `local restart_count = 0`
   - `M.get_job_id()` returns job_id
   - `M.is_running()` returns job_id ~= nil
   - `M.start()`:
     - If job_id exists, return early (already running)
     - Resolve sidecar path: use `debug.getinfo(1, "S").source:sub(2)` to get this file's path, then `vim.fn.fnamemodify(source, ":h:h:h")` to get plugin root, then append `/sidecar/dist/main.js`
     - Check file exists with `vim.fn.filereadable()`, if not, show error and return
     - Call `vim.fn.jobstart({"node", sidecar_path}, { on_stdout = function(_, data, _) require("ts-explorer.rpc").on_data(data) end, on_stderr = function(_, data, _) require("ts-explorer.log").on_stderr(data) end, on_exit = function(_, exit_code, _) job_id = nil; if exit_code ~= 0 then M._handle_crash() end end })`
     - Store returned job_id
   - `M.stop()`:
     - If job_id, call `vim.fn.jobstop(job_id)`, set job_id = nil
     - Call `require("ts-explorer.rpc").reset()`
   - `M.restart()`:
     - Call M.stop(), reset restart_count = 0, call M.start()
   - `M._handle_crash()`:
     - Increment restart_count
     - Call `require("ts-explorer.rpc").reset()`
     - If restart_count <= config max_restarts: call M.start()
     - Else: `vim.notify("TypeScript Explorer: sidecar crashed repeatedly. Use :TsExplorerRestart to retry.", vim.log.levels.ERROR)`

5. **lua/ts-explorer/init.lua** -- Public API:
   - `M.setup(opts)` calls `require("ts-explorer.config").setup(opts)`
   - Return M

6. **plugin/ts-explorer.lua** -- Entry point (runs on plugin load):
   - Guard: `if vim.g.loaded_ts_explorer then return end; vim.g.loaded_ts_explorer = true`
   - Register `:TsExplorerRestart` command -> `require("ts-explorer.sidecar").restart()`
   - Register `VimEnter` autocmd (once=true) -> `require("ts-explorer.sidecar").start()`
   - Register `VimLeavePre` autocmd -> `require("ts-explorer.sidecar").stop()`

7. Update Makefile: add `test` target that runs sidecar tests (Lua linting can be added later).
