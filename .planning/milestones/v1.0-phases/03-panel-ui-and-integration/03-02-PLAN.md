  <action>
Create `lua/ts-explorer/panel.lua` as the main panel module. This is the largest piece of the phase.

**Module state:**
```lua
local state = {
  bufnr = nil,       -- scratch buffer (persists across open/close)
  winid = nil,       -- window handle (nil when closed)
  tree_state = nil,  -- current tree state from tree.lua
  timer = nil,       -- vim.uv debounce timer
  request_id = 0,    -- monotonic counter for request cancellation
  augroup = nil,     -- autocmd group id
}
```

**Public API:**

1. `M.open()`:
   - If `state.winid` is valid, return (already open)
   - Create buffer if needed (`nvim_create_buf(false, true)`), set buffer options: `buftype=nofile`, `bufhidden=hide`, `swapfile=false`, `modifiable=false`, `filetype=tsexplorer`
   - Call `_setup_keymaps()` on new buffer
   - Read config: `require("ts-explorer.config").get()`
   - Open split: `nvim_open_win(state.bufnr, false, { split = config.panel.position, win = 0, width = config.panel.width })`
   - Do NOT set `winfixwidth` (user decision: resizable with Ctrl+W)
   - Set window options: `number=false`, `relativenumber=false`, `signcolumn="no"`, `cursorline=true`, `wrap=false`, `spell=false`
   - Call `_setup_autocmds()` to register CursorHold and WinClosed
   - Immediately trigger a resolve for current cursor position (so panel shows content on open)

2. `M.close()`:
   - If `state.winid` is valid, call `nvim_win_close(state.winid, true)`
   - Set `state.winid = nil`
   - Call `_teardown_autocmds()` to remove CursorHold/WinClosed autocmds
   - Stop and close timer if active

3. `M.toggle()`:
   - If `state.winid` valid -> `M.close()`, else -> `M.open()`

**Keymaps (`_setup_keymaps`):**
Read keymaps from config. Set buffer-local keymaps with `{ buffer = state.bufnr, noremap = true, silent = true }`:
- `config.panel.keymaps.toggle` (default `<CR>`) -> `M._toggle_node()` which calls `tree.toggle(state.tree_state, cursor_line)` then re-renders
- `config.panel.keymaps.expand_recursive` (default `L`) -> calls `tree.expand_recursive(state.tree_state, cursor_line)` then re-renders
- `config.panel.keymaps.collapse_recursive` (default `H`) -> calls `tree.collapse_recursive(state.tree_state, cursor_line)` then re-renders
- `config.panel.keymaps.close` (default `{"q", "<Esc>"}`) -> `M.close()` -- note this is a table, iterate and set each

**Autocmds (`_setup_autocmds` / `_teardown_autocmds`):**
- Create augroup `"TsExplorerPanel"` with `clear = true`
- `CursorHold` on `*.ts,*.tsx` pattern -> `M._on_cursor_hold(ev.buf)`. IMPORTANT: also listen to `CursorMoved` with same debounce for better responsiveness (research pitfall 6: CursorHold depends on updatetime which defaults to 4000ms). Use `CursorMoved` with the 150ms debounce timer as the primary trigger, and `CursorHold` as a fallback.
- `WinClosed` -> if `tonumber(ev.match) == state.winid` then set `state.winid = nil`
- `_teardown_autocmds`: delete the augroup (which clears all its autocmds)

**Debounce + resolve (`_on_cursor_hold`):**
- Guard: return if filetype is not `typescript` or `typescriptreact`
- Guard: return if current buffer is `state.bufnr` (panel buffer)
- Guard: return if `state.winid` is nil or invalid (panel closed)
- Capture `bufnr` and file path NOW (before timer fires)
- Stop existing timer, create if needed via `vim.uv.new_timer()`
- Start timer at 150ms (no repeat): `timer:start(150, 0, vim.schedule_wrap(...))`
- In callback: verify captured bufnr is still current buffer, then call `_resolve_at_cursor(file, bufnr)`

**Resolve (`_resolve_at_cursor(file, bufnr)`):**
- Guard: `sidecar.is_running()` must be true
- Guard: file must not be empty string
- Compute byte offset: `local line = vim.api.nvim_win_get_cursor(0)[1]` and `local col = vim.api.nvim_win_get_cursor(0)[2]`
- `local byte_offset = vim.fn.line2byte(line) - 1 + col` (line2byte is 1-based, TS positions are 0-based)
- Guard: if `byte_offset < 0` then return (empty buffer per pitfall 1)
- Increment `state.request_id`, capture as `my_id`
- Call `rpc.request("resolve", { filePath = file, position = byte_offset }, function(err, result) ... end)`
- In callback: if `my_id ~= state.request_id` then return (cancelled)
- If err, return silently
- If `result` and `result.node` and `result.node ~= vim.NIL`, call `_update_tree(result.node)`
- If result.node is nil/NIL, keep last result (no flicker per user decision)

**Render (`_update_tree(type_node)`):**
- Create new tree state: `state.tree_state = tree.new(type_node)`
- Call `_render()`

**Render helper (`_render()`):**
- Guard: return if `state.tree_state` is nil or `state.winid` is invalid or `state.bufnr` is invalid
- `local result = tree.render(state.tree_state)`
- Set buffer modifiable, call `nvim_buf_set_lines(state.bufnr, 0, -1, false, result.lines)`, set modifiable false

**Wiring -- update `init.lua`:**
No changes needed to init.lua beyond what exists. The `setup()` call already forwards opts to config, and config now has panel defaults. Panel module is lazy-loaded via `require("ts-explorer.panel")` from commands.

**Wiring -- update `plugin/ts-explorer.lua`:**
Add `:TsExplorer` command after the existing `:TsExplorerRestart` command:

```lua
vim.api.nvim_create_user_command("TsExplorer", function()
  require("ts-explorer.panel").toggle()
end, { desc = "Toggle TypeScript Explorer panel" })
```

This keeps the existing VimEnter/VimLeavePre autocmds and TsExplorerRestart command intact.
